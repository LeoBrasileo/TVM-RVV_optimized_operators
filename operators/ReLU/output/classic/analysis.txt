The regression is not in the vector instructions themselves — vfmax.vf is correct and efficient.
The problem is that TVM's lowering pipeline for 4D Relax ops generates a per-axis loop structure that interacts badly with non-power-of-2 inner dimensions:
 the tail handling degenerates into a full separate scalar loop re-entered once per outer row,
 and the vectorization overhead (register pressure, CSR reads, split computation) is paid regardless of whether vectors process any meaningful number of elements.
